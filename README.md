# ai-post-write

Claude Code로 작업한 내용을 **로그로 남기고**, 그 로그를 바탕으로 **AI 활용 사례글**을 작성하는 커맨드 모음입니다.

## 왜 만들었나요?

Claude Code로 뭔가를 만들고 나면 "이거 어떻게 했더라?" 싶을 때가 많습니다. 나중에 사례글을 쓰려고 해도 기억이 안 나고, 대화 내역을 다시 뒤지기도 번거롭죠.

이 커맨드를 쓰면:
1. **작업하면서** `/devlog`로 로그를 남기고
2. **나중에** `/ai-case`로 사례글을 뚝딱 작성할 수 있습니다

---

## 포함된 커맨드

### `/devlog` - 개발 로그 자동 생성

Claude Code와 작업한 내용을 `DEVLOG.md` 파일로 정리해줍니다.

**이런 분께 추천:**
- 작업 내역을 기록으로 남기고 싶은 분
- 나중에 "이거 어떻게 했지?" 찾아보고 싶은 분
- 사례글 작성을 위한 재료를 모아두고 싶은 분

**사용법:**
```
/devlog
```

**결과물 예시:**
- 작업 일시, 사용 도구
- 주요 작업 내역 (뭘 만들었는지)
- 사용한 프롬프트
- 배운 점, 다음에 할 일

---

### `/ai-case` - AI 활용 사례글 작성

`DEVLOG.md`를 바탕으로 비개발자도 읽기 쉬운 사례글을 작성해줍니다.

**특징:**
- **질문을 많이 합니다** - 한 번에 하나씩 질문하며 맥락을 충분히 파악한 뒤 글을 씁니다
- 추천 답변을 제시해서 "응" 한 마디로 진행 가능
- 막혔던 순간, 해결 과정 등 실제 경험 중심으로 작성
- 바로 복붙해서 쓸 수 있는 프롬프트 템플릿 포함
- **커스텀 가능** - 원하는 포맷이나 내용을 추가해달라고 요청하면 반영됩니다

**이런 분께 추천:**
- AI 활용 사례를 팀이나 커뮤니티에 공유하고 싶은 분
- 글쓰기가 막막한 분 (질문에 답하다 보면 글이 완성됨)
- 비개발자 독자를 위한 글을 써야 하는 분

**사용법:**
```
/ai-case
```

**작성 흐름:**
1. DEVLOG 분석 → 어떤 작업을 했는지 파악
2. 맥락 질문 → 문제 상황, 인상적인 순간, 막혔던 부분 등
3. 초안 작성 → `AI_CASE_STUDY.md` 파일 생성
4. 피드백 반영 → 수정 요청하면 바로 반영
5. 마무리 → 이미지 추천 + 게시판 업로드 안내

## 설치 방법

### Mac / Linux

프로젝트 폴더에서 실행:

```bash
curl -fsSL https://raw.githubusercontent.com/daht-mad/ai-post-write/main/install.sh | bash
```

### Windows (PowerShell)

프로젝트 폴더에서 실행:

```powershell
iwr -useb https://raw.githubusercontent.com/daht-mad/ai-post-write/main/install.ps1 | iex
```

### 수동 설치

프로젝트 루트에 `.claude/commands/` 폴더를 만들고 파일을 복사하세요:

```
your-project/
└── .claude/
    └── commands/
        ├── devlog.md
        └── ai-case.md
```

## 권장 워크플로우

```
1. Claude Code로 작업
      ↓
2. /devlog 실행 → DEVLOG.md 생성
      ↓
3. (나중에) /ai-case 실행 → AI_CASE_STUDY.md 생성
      ↓
4. 미리보기 열고 복붙 → 게시판에 업로드
```
